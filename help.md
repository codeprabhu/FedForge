ARCHITECTURE CHECK:

Before we write any code:

1. Explain the problem this component solves.
2. Explain why the architecture needs this component.
3. Explain how industry systems solve the same problem.
4. Explain alternative designs and why we rejected them.
5. Explain every file before creating it.
6. Explain every function before writing it.
7. Explain what would break if we removed it.
8. Only then write code.

Do not optimize for speed.
Optimize for understanding and architecture quality.

Treat FedForge as a distributed systems platform, not an ML tutorial.
